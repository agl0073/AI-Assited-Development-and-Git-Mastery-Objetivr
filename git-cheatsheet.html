🌿 Git Cheat Sheet

Guía de referencia rápida — Proyecto Weather App · AI-Assisted Development

git init branching merge GitHub
⚡ 1 — Inicialización
git initCrea un repositorio Git vacío en la carpeta actual (.git/)
git clone <url>Clona un repositorio remoto existente en local
git config --global user.name "X"Configura tu nombre de autor globalmente
git config --global user.email "x@x"Configura tu email de autor globalmente
📂 2 — Área de Preparación (Stage)
git statusMuestra archivos sin rastrear, modificados o preparados
git add .Añade todos los archivos del directorio al staging
git add <archivo>Añade un archivo específico al staging
git restore --staged <f>Saca un archivo del staging sin perder cambios
💾 3 — Commits
git commit -m "msg"Crea un commit con el mensaje indicado
git commit --amendModifica el último commit (mensaje o archivos)
git diffMuestra las diferencias entre working dir y stage
git diff --stagedDiferencias entre stage y último commit
🌿 4 — Ramas (Branching)
git branchLista todas las ramas locales (asterisco = activa)
git branch <nombre>Crea una nueva rama sin cambiar a ella
git checkout -b <nombre>Crea una rama Y cambia a ella en un solo paso
git checkout <nombre>Cambia a una rama existente
git branch -d <nombre>Elimina una rama ya mergeada
🔀 5 — Fusión (Merge)
git merge <rama>Fusiona la rama indicada en la rama actual
git merge --no-ff <rama>Fuerza un merge commit incluso si es fast-forward
git mergetoolAbre la herramienta para resolver conflictos
git merge --abortCancela un merge en curso con conflictos
☁️ 6 — Repositorio Remoto
git remote add origin <url>Vincula el repo local con un remoto en GitHub
git remote -vLista los remotos configurados y sus URLs
git push -u origin mainSube la rama main al remoto (primera vez)
git pushSube los cambios al remoto (veces siguientes)
git pullDescarga y fusiona cambios del remoto
git fetchDescarga cambios sin fusionar (más seguro)
🔍 7 — Historial (Log)
git logMuestra el historial completo de commits
git log --onelineHistorial resumido: hash corto + mensaje
git log --oneline --graphHistorial con árbol visual de ramas
git show <hash>Muestra los cambios de un commit concreto
↩️ 8 — Deshacer Cambios
git restore <archivo>Descarta cambios no preparados en un archivo
git reset HEAD~1Deshace el último commit manteniendo los cambios
git reset --hard HEAD~1Deshace el último commit y BORRA los cambios
git revert <hash>Crea un commit inverso (seguro para compartido)
💡 9 — Tips Esenciales
.gitignoreArchivo con patrones de rutas a ignorar (ej. /target, *.class)
git stashGuarda cambios temporalmente sin hacer commit
git stash popRecupera los cambios guardados con stash
git tag v1.0Etiqueta un commit como versión (útil para releases)
🌿 Flujo del proyecto: git init → add → commit → branch feature-update → merge → push origin main Weather App · Java + Git CLI + GitHub Pages
